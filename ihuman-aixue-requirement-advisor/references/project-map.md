# 项目地图

在这里沉淀爱学项目的长期结构信息。保持简短，只记录能帮助 Codex 快速定位问题的内容。

## 建议记录

- 仓库包含哪些端或子项目，例如 iOS、Android、Web、后台、SDK、服务端接口定义。
- 主要目录分别负责什么，例如页面、组件、网络层、数据模型、业务服务、路由、资源、测试。
- 常见功能模块的入口，例如课程、直播、练习、作业、账号、支付、会员、班级。
- 重要配置或开关的位置，例如环境配置、灰度开关、feature flag、埋点配置。
- 跨端或跨仓库依赖关系，例如 proto、接口 SDK、公共组件库。

## 维护原则

只记录稳定结构，不记录临时任务结论。发现目录职责变化时，及时更新本文件。
