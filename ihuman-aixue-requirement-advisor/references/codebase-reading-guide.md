# 代码阅读指南

回答爱学项目问题时，按问题类型选择阅读路径。

## 现有功能

1. 搜索用户提到的中文文案、页面标题、接口路径、字段名或枚举值。
2. 找到页面或入口后，沿调用链查看 view/controller/view model/service/network/model。
3. 查看同目录测试、mock、fixture、配置和埋点。
4. 如果有 proto、IDL、OpenAPI 或网络管理层，确认接口字段和状态含义。
5. 最后整理当前行为和代码证据。

## 新需求

1. 先找相邻功能，判断是否已有可复用页面、组件、接口、状态机或数据模型。
2. 找到最小改动路径，优先沿现有架构扩展。
3. 检查是否涉及接口新增、字段兼容、缓存迁移、灰度开关、埋点、权限或多端一致性。
4. 输出实现步骤前，先说明依赖假设和需要确认的业务点。

## 搜索建议

优先搜索：

- 产品中文名、按钮文案、页面标题。
- API 路径、proto message、request/response 字段。
- 常见英文命名、缩写、枚举值。
- 埋点 event、feature flag、配置 key。

如果首次搜索没有结果，尝试同义词、拼音、缩写、大小写变体和历史命名。
