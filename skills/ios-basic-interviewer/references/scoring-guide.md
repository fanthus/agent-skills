# iOS 面试评分维度说明

## 综合评分维度

### 1. 基础扎实度（30%）
- 能否准确描述核心概念（ARC、值/引用类型、Optional、GCD）
- 有没有明显的概念混淆（如 weak/unowned 完全不分）

### 2. 实践经验（30%）
- 能否结合实际项目场景举例
- 遇到过什么坑？怎么解决的？
- 是否用过工具（Instruments、Memory Graph）

### 3. 深度与广度（20%）
- 能否在追问下继续展开（不只是背概念）
- 知识面广度（Swift、UIKit、Concurrency 都有了解）

### 4. 逻辑与表达（20%）
- 解释是否清晰有条理
- 能否用例子辅助说明

---

## 评级参考

| 评级 | 描述 |
|------|------|
| 🌟 强烈推荐 | 基础扎实，有深度，能举一反三，实践经验丰富 |
| ✅ 推荐 | 基础良好，大部分问题回答准确，有一定实践经验 |
| ⚠️ 待定 | 基础有明显短板，但学习能力看起来不错，或某一方向很强 |
| ❌ 不推荐 | 多个核心概念混乱，无法联系实际，基础太薄弱 |
